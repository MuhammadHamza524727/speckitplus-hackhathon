Synthetic Data Generation with Isaac Replicator

Introduction to Isaac Replicator​

Isaac Replicator is NVIDIA's domain randomization tool that generates synthetic datasets with ground truth annotations. It's essential for training perception models without real-world data collection, providing robust datasets for computer vision and robotics algorithms.

Key Features​

  • Domain randomization for robust model training
  • Automatic annotation generation (bounding boxes, segmentation masks, depth maps)
  • Physics-based sensor simulation
  • Integration with Isaac Sim for photorealistic rendering

Installing Isaac Replicator​

Isaac Replicator is included with Isaac Sim but requires additional Python packages:

# Source your Isaac ROS workspace
source ~/isaac_ros_ws/install/setup.bash

# Install Replicator Python dependencies
pip3 install omni.replicator

# Verify installation
python3 -c "import omni.replicator.core as rep; print('Replicator installed successfully')"

# Initialize the replicator

Domain Randomization Configuration​

Domain randomization is the core capability of Isaac Replicator. It creates diverse simulation scenarios by randomizing various scene parameters to generate robust training datasets.

Lighting Randomization​

Lighting randomization helps create datasets that are robust to different illumination conditions:

# Randomize lighting conditions
def randomize_lighting():
    # Create a distant light with randomized properties
    lights = rep.create.light(
        light_type="distant",
        color=rep.distribution.uniform((0.5, 0.5, 0.5), (1.0, 1.0, 1.0)),
        intensity=rep.distribution.normal(3000, 500),
        position=rep.distribution.uniform((-15, -15, 10), (15, 15, 20))
    )

    # Add additional lights for more complex lighting
    additional_lights = rep.create.light(
        light_type="disk",
        color=rep.distribution.uniform((0.7, 0.7, 0.7), (1.0, 1.0, 1.0)),
        intensity=rep.distribution.normal(1500, 300),
        position=rep.distribution.uniform((-5, -5, 5), (5, 5, 10)),
        scale=rep.distribution.uniform((0.5, 0.5, 0.5), (2.0, 2.0, 2.0))
    )

    return lights.node, additional_lights.node

# Register the lighting function
rep.randomizer.register("randomize_lighting", randomize_lighting)

Material and Texture Randomization​

Material randomization creates diverse surface appearances:

# Randomize materials and textures
def randomize_materials():
    # Get all objects in the scene
    objects = rep.get.prims(path_pattern="/World/Objects/*")

    # Randomize materials with different properties
    with objects.randomize("materials", distribution=rep.distribution.uniform(0, 1)):
        # Randomize albedo (base color)
        rep.modify.attribute("xformOp:albedo", rep.distribution.uniform((0.1, 0.1, 0.1), (1.0, 1.0, 1.0)))

        # Randomize roughness for different surface finishes
        rep.modify.attribute("xformOp:roughness", rep.distribution.uniform(0.1, 0.9))

        # Randomize metallic properties
        rep.modify.attribute("xformOp:metallic", rep.distribution.uniform(0.0, 1.0))

        # Apply semantic labels for segmentation
        rep.modify.semantics([("class", rep.distribution.choice(["person", "obstacle", "furniture", "decoration"]))])

    return objects.node

rep.randomizer.register("randomize_materials", randomize_materials)

Environment Randomization​

Randomize the entire environment for comprehensive domain coverage:

# Randomize environment properties
def randomize_environment():
    # Randomize fog density
    fog_settings = rep.create.fog(
        distance_range=rep.distribution.uniform(10, 100),
        height_range=rep.distribution.uniform(0, 5)
    )

    # Randomize atmospheric properties
    atmosphere_settings = rep.create.atmosphere(
        sun_direction=rep.distribution.uniform((-0.5, -0.5, -1), (0.5, 0.5, -0.8)),
        turbidity=rep.distribution.uniform(2.0, 8.0)
    )

    return fog_settings.node, atmosphere_settings.node

rep.randomizer.register("randomize_environment", randomize_environment)

Camera Configuration for Synthetic Data​

Configure camera settings for optimal synthetic data capture:

# Configure camera for data capture
def setup_camera():
    # Create camera prim with realistic parameters
    camera = rep.create.camera(
        position=rep.distribution.uniform((-2, -2, 1), (2, 2, 3)),  # Randomize position
        rotation=rep.distribution.uniform((-10, -10, -180), (10, 10, 180)),  # Randomize rotation
        focal_length=rep.distribution.uniform(18.0, 50.0),  # Randomize focal length
        focus_distance=rep.distribution.uniform(5.0, 50.0),
        horizontal_aperture=20.955,
        vertical_aperture=15.2908
    )

    # Set up camera sensors with multiple data types
    render_product = rep.create.render_product(
        camera.path,
        (1280, 720),  # Resolution
        data_types=[
            "rgb",
            "depth",
            "semantic_segmentation",
            "instance_segmentation",
            "normal",
            "motion_vector"
        ]
    )

    return camera.node, render_product

rep.randomizer.register("setup_camera", setup_camera)

Advanced Randomization Techniques​

For more sophisticated domain randomization:

# Advanced randomization using custom functions
def advanced_randomization():
    # Create a function that randomizes multiple aspects simultaneously
    def randomize_scene():
        # Get all prims in the scene
        prims = rep.get.prims()

        # Randomize different aspects based on prim type
        for prim in prims:
            if "Robot" in prim.name:
                # Special handling for robot models
                rep.modify.pose(
                    position=rep.distribution.uniform((-1, -1, 0), (1, 1, 0)),
                    rotation=rep.distribution.uniform((0, 0, -90), (0, 0, 90))
                )
            elif "Object" in prim.name:
                # Randomize object properties
                rep.modify.attribute("xformOp:scale", rep.distribution.uniform(0.8, 1.2))

    return rep.randomizer.wrap(randomize_scene)

rep.randomizer.register("advanced_randomization", advanced_randomization)

Synthetic Data Generation Workflow​

The complete workflow for generating synthetic datasets with Isaac Replicator involves several key steps:

1. Scene Setup​

First, prepare your Isaac Sim scene with the objects and environment you want to simulate:

import omni.replicator.core as rep
import omni
from pxr import UsdGeom

# Get the current stage
stage = omni.usd.get_context().get_stage()

# Set up the basic scene with objects
# Add ground plane, lighting, and objects to randomize

2. Configure Randomization Graph​

Define what aspects of the scene to randomize:

# Create the randomization graph
with rep.new_layer():
    # Register all the randomization functions we created
    rep.randomizer.randomize_lighting()
    rep.randomizer.randomize_materials()
    rep.randomizer.randomize_environment()
    rep.randomizer.setup_camera()
    rep.randomizer.advanced_randomization()

3. Set Up Data Writers​

Configure how and where the synthetic data will be saved:

# Set up the writer to save generated data
writer = rep.WriterRegistry.get("BasicWriter")
writer.initialize(
    output_dir="path/to/synthetic_dataset",
    rgb=True,
    depth=True,
    semantic_segmentation=True,
    instance_segmentation=True,
    bounding_box_2d_tight=True,
    bounding_box_2d_loose=True,
    # Additional data types
    normal=True,
    motion_vector=True
)

4. Execute Generation​

Run the orchestrator to generate the synthetic dataset:

# Configure the orchestrator
rep.orchestrator.set_num_samples_per_frame(5)  # Generate 5 variations per frame
rep.orchestrator.set_max_num_frames(200)  # Generate 200 * 5 = 1000 total samples

# Attach the writer to the render product
writer.attach([render_product])

# Run the generation
rep.orchestrator.run()

5. Complete Generation Script​

Here's a complete script that puts it all together:

import omni.replicator.core as rep
import numpy as np

# Initialize replicator
rep.orchestrator.init()

# Define randomization functions (as defined earlier)
def randomize_lighting():
    lights = rep.create.light(
        light_type="distant",
        color=rep.distribution.uniform((0.5, 0.5, 0.5), (1.0, 1.0, 1.0)),
        intensity=rep.distribution.normal(3000, 500),
        position=rep.distribution.uniform((-15, -15, 10), (15, 15, 20))
    )
    return lights.node

def setup_camera():
    camera = rep.create.camera(
        position=rep.distribution.uniform((-2, -2, 1), (2, 2, 3)),
        rotation=rep.distribution.uniform((-10, -10, -180), (10, 10, 180)),
        focal_length=rep.distribution.uniform(18.0, 50.0)
    )

    render_product = rep.create.render_product(
        camera.path,
        (1280, 720),
        data_types=["rgb", "depth", "semantic_segmentation"]
    )

    return camera.node, render_product

# Register the functions
rep.randomizer.register("randomize_lighting", randomize_lighting)
rep.randomizer.register("setup_camera", setup_camera)

# Create the randomization graph
with rep.new_layer():
    rep.randomizer.randomize_lighting()

# Set up the writer
writer = rep.WriterRegistry.get("BasicWriter")
writer.initialize(
    output_dir="path/to/synthetic_dataset",
    rgb=True,
    depth=True,
    semantic_segmentation=True
)

# Configure and run the orchestrator
camera_node, render_product = rep.randomizer.setup_camera()
writer.attach([render_product])

rep.orchestrator.set_num_samples_per_frame(3)
rep.orchestrator.set_max_num_frames(100)

# Run the generation
rep.orchestrator.run()

print("Synthetic dataset generation completed!")

Annotation Generation​

Isaac Replicator automatically generates various types of annotations in standard formats:

RGB and Depth Data​

RGB images are saved in standard formats (PNG, JPEG) with pixel values in the range [0, 255]. Depth data is saved as 16-bit grayscale images or EXR files with depth values in meters.

# Generate RGB and depth annotations
writer = rep.WriterRegistry.get("BasicWriter")
writer.initialize(
    output_dir="path/to/output/directory",
    rgb=True,
    depth=True,
    semantic_segmentation=True,
    instance_segmentation=True,
    bounding_box_2d_tight=True,
    bounding_box_2d_loose=True
)
writer.attach([render_product])

Annotation Format Specifications​

Semantic Segmentation​

  • Format: 8-bit indexed PNG
  • Each pixel value represents a class ID
  • Accompanied by a JSON file mapping class IDs to names
  • Example: 
    {0: "background", 1: "person", 2: "obstacle"}

Instance Segmentation​

  • Format: 8-bit indexed PNG
  • Each pixel value represents an instance ID
  • Accompanied by a JSON file with instance metadata
  • Format: 
    {"instances": [{"id": 1, "class": "person", "mask": "path/to/mask.png"}]}

Bounding Boxes​

  • Format: JSON with COCO format
  • Structure: 
    {"bbox": [x, y, width, height], "category_id": class_id, "area": area, "iscrowd": 0}
  • Coordinates in pixels from top-left corner

Depth Maps​

  • Format: 16-bit PNG or EXR
  • Values represent distance from camera in meters
  • Scale factor: 1000 (divide by 1000 to get meters)

# Configure semantic segmentation
def setup_semantic_segmentation():
    # Assign semantic labels to objects
    objects = rep.get.prims(path_pattern="/World/Objects/*")

    # Randomize semantic labels
    with objects.randomize("semantic", distribution=rep.distribution.uniform(0, 1)):
        # Set semantic label based on object type
        rep.modify.semantics([("class", "person")])  # For humanoid robots
        rep.modify.semantics([("class", "obstacle")])  # For static objects
        rep.modify.semantics([("class", "furniture")])  # For furniture

    return objects.node

Isaac Replicator on Jetson/RTX Platforms​

For deployment on edge platforms, optimize Replicator settings:

# For Jetson platforms, use lower resolution for faster processing
export NVX_USE_OPTIMIZED_LIBS=1
export CUDA_DEVICE_ORDER=PCI_BUS_ID
export CUDA_VISIBLE_DEVICES=0
# Optimize Replicator for edge devices
def optimize_replicator_for_edge():
    # Reduce resolution for faster processing
    render_product = rep.create.render_product(
        camera.path,
        (640, 480),  # Lower resolution for edge devices
        data_types=["rgb", "depth"]
    )

    # Limit the number of randomization variations
    rep.orchestrator.set_num_samples_per_frame(1)

    return render_product.node

Performance Optimization​

Batch Processing​

# Configure batch processing for efficiency
def setup_batch_processing():
    # Set the number of samples to generate
    num_samples = 1000

    # Configure the orchestrator
    rep.orchestrator.set_num_samples_per_frame(10)  # Generate 10 samples per frame
    rep.orchestrator.set_max_num_frames(num_samples // 10)  # Calculate total frames needed

    # Enable ADB (Automatic Domain Bridging) for faster processing
    rep.orchestrator.enable_automatic_domain_randomization(True)

Memory Management​

# Optimize memory usage for large datasets
def setup_memory_optimization():
    # Enable streaming for large datasets
    writer = rep.WriterRegistry.get("BasicWriter")
    writer.initialize(
        output_dir="path/to/output/directory",
        rgb=True,
        depth=True,
        semantic_segmentation=True,
        # Use streaming to manage memory for large datasets
        streaming=True,
        # Limit buffer size to prevent memory overflow
        buffer_size=64
    )

Troubleshooting Replicator Issues​

Common Issues and Solutions​

  • "Out of memory" during generation: Reduce resolution or batch size
  • Slow generation speed: Optimize scene complexity or use fewer randomization parameters
  • Missing annotations: Verify sensor setup and writer configuration
  • Inconsistent lighting: Check lighting randomization bounds

Troubleshooting Isaac Platform Integration​

Common Integration Issues with Replicator​

  • Replicator Not Generating Annotations: Verify that the writer is properly initialized and that the render product is attached to the writer
  • Slow Dataset Generation: Reduce the number of randomization parameters or lower the resolution
  • Missing Semantic Segmentation: Ensure semantic labels are properly assigned to objects in the scene
  • Memory Issues with Large Datasets: Enable streaming in the writer configuration and reduce the buffer size

Replicator Hardware Recommendations​

For Small-Scale Data Generation (up to 1K images/day)​

  • GPU: RTX 3060 with 12GB+ VRAM
  • CPU: 6+ cores with good single-core performance
  • RAM: 32GB
  • Storage: 1TB SSD for dataset storage
  • Use Case: Learning, prototyping, small experiments

For Medium-Scale Data Generation (up to 10K images/day)​

  • GPU: RTX 4070 Ti or RTX 3080 with 12GB+ VRAM
  • CPU: 8+ cores with good multi-threading
  • RAM: 64GB
  • Storage: 5TB+ SSD array or NVMe RAID
  • Use Case: Production datasets, research projects

For Large-Scale Data Generation (up to 100K images/day)​

  • GPU: RTX 4080/4090 or RTX 6000 Ada with 24GB+ VRAM
  • CPU: 16+ cores high-performance processor
  • RAM: 128GB+
  • Storage: High-speed NVMe RAID with 10TB+ capacity
  • Use Case: Industrial applications, large model training
